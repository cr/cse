## Interface

### mn_classify (dispatcher, mn_classify.s)
Forwards to `mn6_classify` or `mn7_classify` depending on build-time
`USE_MN6` define.  Default: mn7.

### mn6_classify / mn7_classify
**In:** `mn_c1`, `mn_c2`, `mn_c3` (ZP, VICII screen codes: A=$01..Z=$1A)
**Out:** C=0 → recognized, A = hash slot.  C=1 → unrecognized.
**Clobbers:** A, X, Y, mn*_h_tmp

On success, index `mn_base_op[slot]` and `mn_profile[slot]` for
the base opcode and operand profile.

**Re-exports:** `mn_base_op`, `mn_profile` — aliases to the selected
classifier's tables.

**Depends on:** mn_vars (mn_c1/c2/c3), mn6_tables or mn7_tables (GENERATED)

### Memory

**mn_vars.s ZP (3 bytes):** `mn_c1` (1), `mn_c2` (1), `mn_c3` (1) — mnemonic character inputs.

**mn7.s ZP (1 byte):** `mn7_h_tmp` (1) — holds c3>>2 during fingerprint check.

**mn7.s RODATA (27 bytes):** `mn7_hash_t` — the T[c2] hash perturbation table.

## Variants

|  | mn6 | mn7 |
|--|-----|-----|
| File | mn6.s + mn6_tables.s | mn7.s + mn7_tables.s |
| Mnemonics | 56 (legal NMOS only) | 114 (legal + CMOS + illegal) |
| Hash bits | 6 | 7 |
| Slots | 64 | 128 |
| Hash | `(c1*8 + c3*15 + T[c2]) & $3F` | `(c1*4 + c3 + T[c2]) & $7F` |
| Fingerprint | `(c1 + c2*218) & $FF` | `(c2<<3) \| (c3>>2)` |
| False positives | 16 / 17576 (0.09%) | 0 / 17576 |
| Table size | 3×64 + 27 + 27 = 246 B | 3×128 + 27 = 411 B |
| Build flag | `-D USE_MN6` | default |

Both use the same table layout per slot: fingerprint (1B),
base_op (1B), profile (1B).  Both use a 27-byte T[c2] lookup in
RODATA (index 0 = guard, 1–26 = A–Z).

## Design

1. Compute hash from the three screen-code characters
2. Index the fingerprint table at the hash slot
3. Compare computed fingerprint against stored fingerprint
4. Match → C=0, return slot in A.  Mismatch → C=1.

The fingerprint check eliminates false positives (mn7 has zero;
mn6 has 16, all requiring two simultaneous typos from any legal
mnemonic).

## Caveats

- Both classifiers clobber Y.  Caller must `ldy #0` after the call
  if Y is needed.
- VICII screen codes are 1-based (A=1..Z=26).  Index 0 in T[] is
  a guard value ($FF).
- mn6 does not recognize CMOS or illegal mnemonics.  Use mn7 for
  6510 illegal opcode support or 65C02 CMOS support.
- Tables are GENERATED by `dev/mnemonic_tables.py`.  Do not edit
  mn6_tables.s or mn7_tables.s by hand.
