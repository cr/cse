# zp.s — Central Zero-Page Layout

**Template:** [module](../templates/module.md)

## Owned files

| File | Role |
|------|------|
| [`src/zp.s`](../../src/zp.s) | implementation — single source of truth for all ZP allocations |

## Interface

Every ZP variable in the project is defined and `.exportzp`'d by zp.s.
Modules `.importzp` what they need.  No other file may contain a
`.segment "ZEROPAGE"` section in production code.

### Exported symbols (~92 bytes, $02–$5D)

| Range | Consumer | Variables |
|-------|----------|-----------|

| $08 | asm_err | `_asm_saved_sp` (1) |
| $09–$20 | assembler | `asm_pc` (2), `asm_out` (2), `asm_cpu` (1), `asm_len` (1), `asm_slot` (1), `asm_prof` (1), `asm_pidx` (1), `asm_base` (1), `asm_bit` (1), `asm_mode` (1), `asm_tmp` (1), `asm_tmp2` (1), `sym_name` (2), `sym_val` (2), `sym_wide` (1), `expr_ptr` (2), `expr_val` (2), `expr_wide` (1) |
| $21–$23 | asm_src | `_as_ptr` (2), `_as_wsize` (1) |
| $24–$26 | mn_vars | `mn_c1` (1), `mn_c2` (1), `mn_c3` (1) |

First free byte: $5E.  User programs may use $5E–$7F (34 bytes).
`__ZP_LAST__` is auto-generated by the linker (`define=yes` on the
ZP memory region in the linker config).

### Cross-module flag semantics

Seven 1-byte state values are hosted here because they are read
across module boundaries, typically by a lower layer than the
module that writes them.  Hosting them in zp.s eliminates the
back-edges that would otherwise arise.

| Flag | Writer (semantic owner) | Readers |
|---|---|---|
| `in_userland` | debugger.s (set), main.s (clear) | main.s (cse_nmi_handler dispatch) |
| `state` | main.s, editor.s | editor.s, repl.s (mode checks) |
| `warm_cont` | repl.s gates (`a`/`l`/`R`) | main.s (main_loop_top) |
| `kernal_out` | asm_src.s (batch discipline), mem.s | mem.s (bank helpers), symtab.s |
| `ed_dirty` | editor.s | repl.s (unsaved-changes gates) |
| `dbg_reason` | debugger.s (BRK handler) | repl.s (gates, `c` command), main.s |
| `cur_device` | repl.s (`$ N`) | disk.s |

Rule of thumb: a flag migrates to zp.s when at least one reader
sits in a strictly lower layer than the writer.  Flags with a
single reader in the same or higher layer stay in their module's
BSS.

## Design

ZP is a global resource.  Having each module define its own ZP
segment made the layout implicit and fragile — reordering link
objects or adding a byte shifted every downstream address.

zp.s solves this: one file, sequential `.res` allocations, explicit
address annotations in comments.  The linker assigns addresses from
$02 upward; the annotations are verified against the linker map.

### Naming convention

- `asm_*`, `sym_*`, `expr_*` — shared I/O (set by callers, read by
  callees, or vice versa)
- `_asm_*`, `_io_*`, `_st_*`, `_ex_*`, `_dasm_*`, `_as_*` — single-consumer
  scratch (underscore prefix signals "only one module writes this")
- `rp_*` — runtime scratch pointers shared by repl, debugger, mem
- `mn_c1`..`mn_c3` — mnemonic classifier inputs
- `gap_*`, `ed_*`, `buf_base` — editor state

### Test stubs

Test binaries link zp.s for all ZP definitions.  Stubs no longer
need to duplicate `.exportzp` / `.res` blocks.  Exception: test-only
indirect pointers that need ZP addressing (e.g. `_src_ptr` in
`asm_src_test_stub.s`) may define private ZP vars.

## Caveats

- **Link order matters**: zp.s must be linked first so it owns the
  ZEROPAGE segment.  The Makefile lists `zp` first in `ASM_SRCS`.
- **mn6/mn7 alias**: `mn7_h_tmp` and `mn6_h_tmp` share address $27.
  Only one classifier is linked per build (selected by `USE_MN6`).
- **Address annotations**: The `── $XX` comments in zp.s are
  informational, derived from the linker map.  If the layout changes,
  regenerate by checking `build/*/trial.map`.
