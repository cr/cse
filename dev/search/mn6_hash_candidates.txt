Generated by deterministic co-optimisation (dev/clean_hash_search.py).
Hash formula:   h  = (c1*C1 + c3*C3 + T[c2]) & $3F
Fingerprint:    fp = (c1*A  + c2*B)  & $FF
Characters:     VICII screencodes, A=1 .. Z=26.
Search space:   C1, C3 in 1..63; A, B in 0..255.
Valid hashes:   1264 (C1,C3) pairs admit a collision-free T table.
Ceiling:        min_wdl = 1.5 is the maximum achievable over all 1264 pairs.
                113 pairs achieve it.  No hash can push all false positives
                beyond wdist 1.5 from every legal mnemonic.

QWERTY adjacency (weighted Hamming distance):
  0.0  same character
  0.5  physically adjacent keys (includes cross-row diagonals: E<->D, R<->F, ...)
  1.0  all other pairs
  Range 0.0 .. 3.0 over 3-character strings.

"min_wdl" = minimum wdist(fp_string, nearest_legal) over all false positives.
Higher is better.  1.5 is the proven ceiling.

All T tables produced by the deterministic bitmask backtracking solver
(largest-c2-group first, ascending T values), giving reproducible results.

================================================================================
SEARCH METHODOLOGY
================================================================================

Tool:    dev/clean_hash_search.py
Runtime: ~7 minutes on Apple Silicon (numpy vectorised fingerprint search,
         bitmask backtracking T-solver).

T-solver: deterministic iterative backtracking with 64-bit occupancy bitmasks.
  Groups mnemonics by c2; processes largest group first (most constrained).
  Within each group tries T values 0..63 in ascending order.
  Produces the lexicographically first valid T table for each (C1,C3).

Fingerprint search: for each valid (C1,C3,T), finds best (A,B) in 0..255^2.
  Groups non-legal candidates by (delta1,delta2) = (cand_c1-legal_c1, cand_c2-legal_c2).
  For each unique pair, computes a 256x256 boolean solution matrix via numpy
  broadcast.  Selects (A,B) maximising min_wdl, then minimising FP count.
